# LLM 能力边界测试方案

## 产品背景

可穿戴 AI 相机（类似 Looki L1），核心能力：
- 定时拍照（动态频率）
- AI 判断场景重要性 → 调整拍照频率 / 触发视频录制
- 事后回顾和搜索

## 待测模型

| 模型 | 能力 | 产品角色 |
|------|------|---------|
| Sonnet 4.6 | 图像理解 | 实时判断：场景是否重要，是否触发录制 |
| Kimi K2.5 | 视频理解 | 后处理：回看视频，提取高光，判断录制质量 |

## 核心架构：两级决策模型

```
[持续低频拍照] → 图像模型判断 → 重要？→ 提高频率 / 触发录制
                                         ↓
                              [录制中] → 图像模型持续监测 → 无聊N帧？→ 停止录制
                                         ↓
                              [录制完成] → 视频模型后处理 → 提取高光 / 生成摘要
```

### 关键决策点

1. **何时开始录制** — 图像模型（单图分类，低延迟）
2. **录多久 / 何时停止** — 图像模型（连续帧监测，状态机驱动）
3. **录的内容有没有价值** — 视频模型（后处理，非实时）

> 视频模型不参与实时决策。原因：延迟太高，不适合在线判断。
> 视频模型的价值在"录完之后"：剪辑、摘要、高光提取、质量评估。

## 第一轮：单图场景分类（Sonnet 4.6）

### 目标
验证模型能否从单张可穿戴相机照片判断场景重要性。

### 数据准备
- 100-200 张图片
- 覆盖场景：

| 类别 | 有意义示例 | 无意义示例 |
|------|-----------|-----------|
| 社交 | 聚会、和人交谈 | 空房间 |
| 环境 | 新地方、旅行风景 | 同一个办公桌 |
| 事件 | 运动、表演、美食 | 走路、等电梯 |
| 信息 | 白板、路标、名片 | 模糊/遮挡画面 |

- 标注格式：

```json
{
  "image_id": "001",
  "file": "photos/001.jpg",
  "label": "important",
  "importance_score": 4,
  "category": "social",
  "description": "朋友生日聚会，正在切蛋糕"
}
```

### 测试 Prompt（v1 基线）

```text
你是一个可穿戴相机的 AI 判断模块。用户将这个相机别在胸前，它每隔几秒拍一张照片。
你的任务是判断这张照片是否捕捉到了"值得记录"的瞬间。

判断标准：
- 社交互动（与人交流、聚会）
- 新环境/新场景（旅行、第一次去的地方）
- 值得回忆的瞬间（美食、美景、有趣的事）
- 重要信息（白板内容、路标、名片）

请以 JSON 格式输出：
{
  "importance_score": 1-5,
  "should_increase_frequency": true/false,
  "should_start_recording": true/false,
  "reason": "一句话解释"
}
```

### 评测指标
- 准确率（与人工标注的一致性）
- **误报率**（无聊→重要，耗电杀手，重点关注）
- **漏报率**（重要→无聊，用户体验杀手）
- 响应延迟（ms）
- 不同 prompt 版本对比

## 第二轮：连续帧状态转换检测（Sonnet 4.6）

### 目标
验证模型能否从连续 3-5 张图片中检测"场景变化"，为录制启停提供依据。

### 数据准备
- 30-50 组图片序列，每组 3-5 张（模拟每 5 秒拍一张）
- 有转换组：室内→室外、独处→社交、静止→运动
- 无转换组：持续办公、持续步行、持续看手机

### 测试 Prompt

```text
以下是可穿戴相机在最近 15 秒内拍摄的连续照片（每 5 秒一张，按时间顺序排列）。

请判断：
1. scene_changed: 场景是否发生了显著变化？(true/false)
2. change_type: 如果有变化，变化的性质（环境切换/人物出入/活动变化/无变化）
3. action: 建议的相机行为（maintain/increase_frequency/start_recording/stop_recording）
4. reason: 一句话解释
```

### 评测重点
- 多图输入的推理质量
- 时序理解能力（能否理解"变化方向"）
- 与单图方案的增量价值

## 第三轮：视频理解（Kimi K2.5）

### 目标
验证视频模型的后处理能力：高光提取、内容摘要、录制质量评估。

### 数据准备
- 20-30 段 10-30 秒短视频
- 有意义：朋友讲笑话、街头表演、日落
- 无意义：走路、等电梯、刷手机
- 标注：重要性评分 + 关键时间点

### 测试任务

**任务 A：视频价值评估**
```text
这是一段可穿戴相机录制的视频。请评估：
1. overall_value: 1-5（值得保留的程度）
2. highlights: 视频中最有价值的时间段（起止秒数）
3. summary: 一句话描述视频内容
4. keep_or_discard: 建议保留还是丢弃
```

**任务 B：录制时长反馈（为未来优化提供数据）**
```text
回看这段视频，如果你在第 0 秒时决定开始录制：
1. optimal_duration: 理想情况下应该录多少秒？
2. wasted_seconds: 有多少秒是"无用"的？
3. event_end_time: 有意义的事件在第几秒结束？
```

### 评测重点
- 视频事件理解 vs 仅看画面
- 高光定位准确性
- 响应延迟（影响后处理体验）

## 录制时长控制策略

### 问题本质
"拍多久"不是视频模型实时决定的，而是一个状态机问题。

### 方案：图像模型驱动的状态机

```
状态: IDLE（低频拍照，如每10秒一张）
  ↓ 图像模型判断 importance_score >= 4
状态: ALERT（高频拍照，如每2秒一张）
  ↓ 连续3帧 importance_score >= 4
状态: RECORDING（开始录制视频）
  ↓ 连续N帧 importance_score <= 2（"冷却"）
状态: COOLDOWN（继续录制，但开始倒计时）
  ↓ 冷却时间到（如10秒）
状态: IDLE（停止录制，保存视频）
```

### 关键参数（需通过测试确定）
- `ALERT_THRESHOLD`: 触发高频拍照的 importance_score（初始值: 4）
- `RECORDING_THRESHOLD`: 触发录制的连续高分帧数（初始值: 3）
- `COOLDOWN_FRAMES`: 低分多少帧后停止录制（初始值: 5）
- `COOLDOWN_TIME`: 冷却等待时间（初始值: 10秒）
- `MAX_RECORDING_TIME`: 单次录制上限（初始值: 120秒，防止失控）

### 视频模型的角色（后处理）
录制结束后，视频模型负责：
1. 评估录制质量 → 反馈给状态机参数调优
2. 提取高光片段 → 用户只看精华
3. 判断"应该录更久/更短" → 离线优化冷却参数

## 执行顺序

```
第1步: 收集和标注图片/视频数据（1-2天）
第2步: 第一轮 Sonnet 4.6 单图分类 → 基线能力
第3步: 迭代 prompt → 找最优版本
第4步: 第二轮 Sonnet 4.6 连续帧 → 多图推理能力
第5步: 第三轮 Kimi K2.5 视频理解 → 后处理能力
第6步: 对比分析 → 确定产品架构
```

## 成功标准

| 指标 | 最低要求 | 理想目标 |
|------|---------|---------|
| 单图分类准确率 | > 80% | > 90% |
| 误报率（无聊→重要） | < 15% | < 5% |
| 漏报率（重要→无聊） | < 10% | < 5% |
| 单图响应延迟 | < 3s | < 1s |
| 视频高光定位误差 | < 5s | < 2s |
